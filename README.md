# SIMD Beast Mode: Разгоняем Матрицы на Полную

## Что за зверь?

Этот проект - твой личный ускоритель для математических операций в C++. Мы берем SIMD (Single Instruction, Multiple Data) инструкции и жмем их до упора. Тестируем базовые реализации, SSE и AVX оптимизации для матричного умножения и всякой такой фигни. Короче, если хочешь, чтобы твои вычисления летали быстрее, чем ты успеваешь моргнуть - это сюда.

## Что внутри?

- `src/main.cpp` - Тут вся магия на C++, алгоритмы, которые заставят твой процессор вспотеть
- `main.py` - Python-скрипт, который соберет результаты и вывалит их в Excel, как будто это твое личное казино
- `Makefile` - Сборка, потому что вручную компилировать - для слабаков
- `bin/` - Тут живут исполняемые файлы и результаты, не суйся без спроса

## Какие операции мы рвем?

Программа поддерживает 11 вариантов математических фокусов, от простых до тех, что заставят тебя почесать затылок:

1. A * x - классика
2. a * (A * x) + b * (B * y) - для тех, кто любит сложности
3. A * x + b * (B * y) - баланс между простотой и мощью
4. a * (A * x) + B * y - когда нужно все и сразу
5. a * (A * x) + y - почти просто, но с изюминкой
6. a * x + b * (B * y) - для любителей симметрии
7. a * (A * x) - b * (B * y) - минус тоже бывает полезен
8. A * x - b * (B * y) - комбо из плюса и минуса
9. a * (A * x) - B * y - вычитаем матрицы
10. a * (A * x) - y - почти как 5, но с минусом
11. a * x - b * (B * y) - финальный босс простоты

## Типы данных - выбирай свой яд

Три типа, как три дороги в жизни:
- int - целые числа, для тех, кто считает копейки
- float - плавающая точка одинарной точности, быстрый и грязный
- double - двойная точность, для перфекционистов, которые не верят в компромиссы

## Оптимизации - от нуля до героя

Для каждого типа и операции мы даем три уровня прокачки:
- **Base** - Базовый уровень, без SIMD, как бегать пешком
- **SSE** - SSE инструкции, первый буст скорости
- **AVX** - AVX на максимум, когда нужно разогнаться до космических скоростей

## Как запустить эту ракету?

### Требования
Для сборки проекта нужен Microsoft Visual C++ компилятор (cl.exe), который входит в состав Visual Studio или Build Tools for Visual Studio. Убедись, что переменная окружения настроена правильно.

### Сборка
1. Открой командную строку (cmd.exe) или PowerShell.
2. Перейди в корневую папку проекта: `cd H:\Inbox\Downloads\Architecture_and_software_2`
3. Если нужно, отредактируй Makefile и укажи правильный путь к `vcvars64.bat` в переменной VCVARS (обычно в "C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvars64.bat" или аналогично для твоей версии VS).
4. Запусти сборку:
   ```
   nmake /f Makefile
   ```
   Это скомпилирует `main.cpp` с помощью cl.exe и создаст исполняемый файл в `bin/main.exe`.

### Запуск
После сборки запусти программу:
```
bin\main.exe
```
Результаты будут сохранены в папке `results/`, а Python-скрипт `main.py` может обработать их для анализа.

### Запуск
```bash
./bin/main.exe
```
Войди в меню, выбери, что тестировать, и смотри, как цифры летят.

## Что делать с результатами?

После тестов результаты улетают в `results/`. Запусти `main.py`, и он превратит их в красивые Excel-таблицы с графиками производительности.

```python
python main.py
```
Готово, теперь ты знаешь, кто чемпион.

## Под капотом

Мы используем Intel интринсики для прямого общения с SIMD:
- SSE: `__m128`, `__m128i` - для 4-элементных векторов
- AVX: `__m256`, `__m256i` - для 8-элементных, потому что почему бы и нет

Код заточен под вектора фиксированного размера (4 для SSE, 8 для AVX) и горизонтальные операции для суммирования. Никаких компромиссов, только скорость (lol).
